# **Input-Output and Interrupt Instructions** 

---

## **1\. Introduction**

Input-Output (I/O) and Interrupt Instructions are **special types of instructions used for communication between CPU and external devices**.

* CPU interacts with **input devices** (keyboard, sensors) and **output devices** (monitor, printer).

* **Interrupt instructions** allow the CPU to temporarily **stop execution** and handle urgent events.

üìå These instructions are essential for **I/O operations and multitasking**.

---

## **2\. Classification**

I/O instructions are mainly classified into:

| Type | Description |
| ----- | ----- |
| **Input Instructions** | Read data from input device into CPU register (usually AC) |
| **Output Instructions** | Send data from CPU register (usually AC) to output device |
| **Control/Interrupt Instructions** | Enable, disable, or handle interrupts |

---

## **3\. Format of I/O Instructions (Basic Computer ‚Äì Mano)**

* 16-bit instruction word

* Opcode = 111

* I-bit = 1 (to differentiate from Register-Reference Instructions)

`| I | Opcode | Address |`

* **I = 1 ‚Üí I/O Instruction**

* Address field specifies I/O device or control operation.

---

## **4\. Common I/O Instructions in Basic Computer**

| Instruction | Function | Description |
| ----- | ----- | ----- |
| **INP** | Input | Transfer data from input device to AC |
| **OUT** | Output | Transfer data from AC to output device |
| **SKI** | Skip on Input | Skip next instruction if input flag is set |
| **SKO** | Skip on Output | Skip next instruction if output flag is set |
| **ION** | Interrupt ON | Enable interrupts |
| **IOF** | Interrupt OFF | Disable interrupts |

---

## **5\. Micro-Operations of I/O Instructions**

### **A. Input Instruction (INP)**

* AC(0‚Äì7) ‚Üê INPR

* INPR = Input Register holding data from input device

**Steps:**

| Time | Micro-Operation |
| ----- | ----- |
| T3 | AC ‚Üê INPR |

---

### **B. Output Instruction (OUT)**

* OUTR ‚Üê AC(0‚Äì7)

* Sends data to output device

**Steps:**

| Time | Micro-Operation |
| ----- | ----- |
| T3 | OUTR ‚Üê AC |

---

### **C. Skip on Input (SKI)**

* Skips next instruction if **input flag (FGI) = 1**

**Micro-Operation:**

`If FGI = 1 then PC ‚Üê PC + 1`

---

### **D. Skip on Output (SKO)**

* Skips next instruction if **output flag (FGO) = 1**

**Micro-Operation:**

`If FGO = 1 then PC ‚Üê PC + 1`

---

### **E. Interrupt ON (ION)**

* Enables CPU to respond to **hardware interrupts**

**Micro-Operation:**

`Set IEN (Interrupt Enable Flag) ‚Üê 1`

---

### **F. Interrupt OFF (IOF)**

* Disables hardware interrupts

**Micro-Operation:**

`Set IEN ‚Üê 0`

---

## **6\. Interrupt Concept**

**Interrupt** is a signal sent by I/O devices or software to CPU, requesting **immediate attention**.

* CPU **saves current execution state**

* Executes **Interrupt Service Routine (ISR)**

* Returns to normal execution

---

### **Types of Interrupts**

1. **Hardware Interrupts** ‚Äì Generated by I/O devices (keyboard, printer, etc.)

2. **Software Interrupts** ‚Äì Generated by programs using I/O instructions

---

### **Interrupt Cycle**

1. CPU **completes current instruction**

2. **PC is saved** in memory or stack

3. **PC loaded with ISR address**

4. Interrupt routine executes

5. Original program resumes from saved PC

---

## **7\. Timing Diagram for I/O Instructions**

**Input (INP) Example:**

`T0 ‚Üí AR ‚Üê PC`  
`T1 ‚Üí IR ‚Üê M[AR], PC ‚Üê PC + 1`  
`T2 ‚Üí Decode IR (I/O instruction)`  
`T3 ‚Üí AC ‚Üê INPR (Data from input device)`

**Output (OUT) Example:**

`T0 ‚Üí AR ‚Üê PC`  
`T1 ‚Üí IR ‚Üê M[AR], PC ‚Üê PC + 1`  
`T2 ‚Üí Decode IR`  
`T3 ‚Üí OUTR ‚Üê AC (Data to output device)`

---

## **8\. Registers Used in I/O Instructions**

| Register | Function |
| ----- | ----- |
| **AC** | Accumulator, holds data for I/O operation |
| **INPR** | Input Register, holds incoming data |
| **OUTR** | Output Register, holds data to be sent |
| **PC** | Program Counter |
| **IR** | Instruction Register |
| **IEN** | Interrupt Enable Flag |
| **FGI / FGO** | Input/Output device flags |

---

## **9\. Summary of I/O Instruction Functions**

| Instruction | Function |
| ----- | ----- |
| INP | Input data to AC |
| OUT | Output data from AC |
| SKI | Skip next instruction if input ready |
| SKO | Skip next instruction if output ready |
| ION | Enable interrupts |
| IOF | Disable interrupts |

---

## **10\. Importance of I/O and Interrupt Instructions**

* Enable CPU to communicate with external devices

* Allow **asynchronous processing**

* Improve **CPU efficiency** (CPU can continue other tasks while waiting for I/O)

* Support **multitasking and real-time processing**

---

## **11\. Advantages**

* Efficient handling of I/O devices

* Reduces CPU idle time

* Supports hardware and software interrupts

* Enables conditional execution based on device status

---

## **12\. Summary**

* I/O instructions manage **data transfer** between CPU and I/O devices

* Interrupt instructions **enable or disable interrupts**

* Controlled by **flags, registers, and micro-operations**

* Form the basis for **asynchronous I/O and real-time system design**
