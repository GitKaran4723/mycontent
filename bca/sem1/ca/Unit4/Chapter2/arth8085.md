# **Arithmetic Operations in 8085 Microprocessor**

---

## **1\. Introduction**

Arithmetic operations in 8085 are instructions that **perform mathematical calculations** such as:

* Addition

* Subtraction

* Increment / Decrement

* Decimal Adjust

* 16-bit addition

**Purpose:**

To process numeric data stored in registers or memory and store the result in the accumulator or register pair.

**Flags Affected:**

* **Sign (S)** – 1 if result is negative, 0 if positive

* **Zero (Z)** – 1 if result is 0

* **Parity (P)** – 1 if even parity, 0 if odd parity

* **Carry (CY)** – 1 if carry out of MSB

* **Auxiliary Carry (AC)** – 1 if carry from bit 3 to bit 4

---

## **2\. Categories of Arithmetic Operations**

Arithmetic instructions in 8085 are divided into **5 types**:

1. **Addition**

2. **Subtraction**

3. **Increment / Decrement**

4. **Add / Subtract with Carry**

5. **Decimal Adjust (BCD)**

---

## **3\. Detailed Explanation of Each Type**

### **3.1 Addition**

* Adds **contents of a register or memory** to the **accumulator** and stores the result in **accumulator**.

**Instructions:**

| Instruction | Description | Operand |
| ----- | ----- | ----- |
| `ADD R` | Add register R to accumulator | R = A, B, C, D, E, H, L, M |
| `ADI data` | Add immediate data to accumulator | 8-bit data |
| `DAD RP` | Add 16-bit register pair to H-L pair | RP = BC, DE, HL, SP |

**Example:**

`MVI A, 25H`  
`MVI B, 35H`  
`ADD B       ; A = A + B = 25H + 35H = 5AH`

* **DAD instruction** adds **16-bit values** and stores the result in **H-L pair**.

---

### **3.2 Subtraction**

* Subtracts **register/memory content** from **accumulator** and stores the result in **accumulator**.

**Instructions:**

| Instruction | Description |
| ----- | ----- |
| `SUB R` | Subtract register R from accumulator |
| `SUI data` | Subtract immediate data from accumulator |
| `SBB R` | Subtract register R \+ carry from accumulator |

**Example:**

`MVI A, 50H`  
`MVI B, 20H`  
`SUB B       ; A = 50H - 20H = 30H`

---

### **3.3 Increment / Decrement**

* Increases or decreases **register/memory** by 1\.

**Instructions:**

| Instruction | Description |
| ----- | ----- |
| `INR R` | Increment register R by 1 |
| `DCR R` | Decrement register R by 1 |
| `INX RP` | Increment register pair RP by 1 |
| `DCX RP` | Decrement register pair RP by 1 |

**Example:**

`MVI B, 0AH`  
`INR B       ; B = B + 1 = 0BH`  
`DCX H       ; H-L pair = H-L - 1`

* **Flags affected:** INR / DCR affect **S, Z, P, AC** (not carry).

---

### **3.4 Add / Subtract with Carry**

* These instructions **include the carry flag** in arithmetic operations.

**Instructions:**

| Instruction | Description |
| ----- | ----- |
| `ADC R` | Add register R \+ carry to accumulator |
| `ACI data` | Add immediate data \+ carry to accumulator |
| `SBB R` | Subtract register R \+ carry from accumulator |
| `SBI data` | Subtract immediate data \+ borrow from accumulator |

**Example:**

`; Assuming CY = 1`  
`MVI A, 20H`  
`MVI B, 30H`  
`ADC B       ; A = 20H + 30H + 1 = 51H`

---

### **3.5 Decimal Adjust (BCD Arithmetic)**

* **DAA (Decimal Adjust Accumulator)** is used after **adding two BCD numbers**.

**Purpose:**

Adjust the result in accumulator to a valid **BCD format**.

**Example:**

`MVI A, 27H`  
`ADI 35H     ; A = 27H + 35H = 5CH (BCD)`  
`DAA         ; Adjust A to BCD = 62H`

---

## **4\. Arithmetic Operations with Memory**

* **Memory location referred by H-L pair** can be used in arithmetic operations.

**Examples:**

`ADD M      ; Add memory content pointed by H-L to accumulator`  
`SUB M      ; Subtract memory content pointed by H-L from accumulator`  
`INR M      ; Increment memory content pointed by H-L`  
`DCR M      ; Decrement memory content pointed by H-L`

---

## **5\. Flags Affected by Arithmetic Instructions**

| Instruction Type | Flags Affected |
| ----- | ----- |
| ADD, ADC, ADI, DAD | S, Z, P, CY, AC |
| SUB, SBB, SUI, SBI | S, Z, P, CY, AC |
| INR, DCR | S, Z, P, AC (CY unchanged) |
| DAD (16-bit) | CY only |
| DAA | S, Z, P, CY, AC |

---

## **6\. Summary Table of Arithmetic Instructions**

| Instruction | Operation | Operand | Example |
| ----- | ----- | ----- | ----- |
| ADD R | A = A \+ R | Register | `ADD B` |
| ADI data | A = A \+ data | Immediate | `ADI 25H` |
| ADC R | A = A \+ R \+ CY | Register | `ADC C` |
| ACI data | A = A \+ data \+ CY | Immediate | `ACI 10H` |
| SUB R | A = A \- R | Register | `SUB D` |
| SUI data | A = A \- data | Immediate | `SUI 20H` |
| SBB R | A = A \- R \- CY | Register | `SBB E` |
| SBI data | A = A \- data \- CY | Immediate | `SBI 05H` |
| INR R | R = R \+ 1 | Register/Memory | `INR B` |
| DCR R | R = R \- 1 | Register/Memory | `DCR C` |
| INX RP | RP = RP \+ 1 | Register Pair | `INX H` |
| DCX RP | RP = RP \- 1 | Register Pair | `DCX D` |
| DAD RP | HL = HL \+ RP | Register Pair | `DAD B` |
| DAA | Decimal adjust accumulator | Accumulator | `DAA` |

---

## **7\. Key Points**

1. Arithmetic operations in 8085 are **accumulator-based**, except register pair operations like **DAD, INX, DCX**.

2. **Carry flag** is used in multi-byte operations or add/subtract with carry.

3. Memory location (pointed by **H-L**) can be used in addition/subtraction and increment/decrement.

4. Decimal Adjust (`DAA`) is essential when working with **BCD numbers**.

5. Proper use of **flags** helps in conditional operations after arithmetic.
